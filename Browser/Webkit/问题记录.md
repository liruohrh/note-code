## 解决方法

Source/WebKit/NetworkProcess/glib/NetworkMDNSRegisterGLib.cpp
NetworkMDNSRegister::registerMDNSName
request 是 unique_ptr。C++ 不规定函数实参求值顺序;GCC 右到左 → 先 request.release()(置空)→ 再 request->cancellable.get() 解引用空指针，导致崩溃。

里面有几个都这样，修复方法：提到前面求值再传递参数。


- webkit仓库修复于 2026-02-17 08:56:04 -0800  [[GLib] NetworkProcess crashes navigating to Twitch](https://bugs.webkit.org/show_bug.cgi?id=308051)



# librice api不兼容
- **`Source/WebCore/platform/rice/GRefPtrRice.h`** 用 `rice_turn_config_ref` / `rice_turn_config_unref` —— librice 在 **v0.3.0 之后删掉了这对 ref/unref API**(改成 copy/free)。→ **v0.4.x 编译失败**
	- https://github.com/ystreet/librice/blob/main/rice-c/src/bindings.rs
	- https://github.com/ystreet/librice/blob/0.3/rice-c/src/bindings.rs


- 将librice 改为v0.2.1(  rice_tls_config_ref ) 
- 或者  禁用rice  USE_LIBRICE
	- 因为访问browerscan主页和ip页崩溃，看起来rice库在不同环境下有点问题，用老库libnice也没什么

手动更新环境的方法
```bash
git clone https://github.com/ystreet/librice
cd librice
git checkout v0.2.1

TRIPLE=$(gcc -dumpmachine)
SUDO_ENV='sudo env PATH=/opt/rust/bin:/usr/local/bin:/usr/bin:/bin RUSTUP_HOME=/opt/rust CARGO_HOME=/opt/rust RUSTUP_TOOLCHAIN=1.94.0'
$SUDO_ENV cargo cinstall -p rice-proto --release --prefix=/usr --libdir=lib/$TRIPLE --library-type=cdylib
$SUDO_ENV cargo cinstall -p rice-io   --release --prefix=/usr --libdir=lib/$TRIPLE --library-type=cdylib

# 清旧版残留 + 刷链接缓存   
# 因为webkit不指定库的版本，要避免用到其他版本
sudo rm -f /usr/lib/$TRIPLE/librice-*.so.0.4.3
sudo ldconfig
# 验证
pkg-config --modversion rice-proto          # 应为 0.2.1
grep -n rice_turn_config_ref /usr/include/rice/rice-proto.h   # 应能找到
```

## 解决方法

Tools\Playwright\win\CMakeLists.txt
```
# WTF and bmalloc symbols live in JavaScriptCore.dll on Windows.  The
# STATICALLY_LINKED_WITH_WTF / STATICALLY_LINKED_WITH_bmalloc definitions
# propagated by WebKit::WTF cause WTF_EXPORT_PRIVATE to expand to
# __declspec(dllexport) instead of __declspec(dllimport), which prevents
# lld-link from resolving the symbols via the JavaScriptCore import library.
# Override them here so the correct dllimport decoration is used.
target_compile_options(Playwright PRIVATE -USTATICALLY_LINKED_WITH_WTF -USTATICALLY_LINKED_WITH_bmalloc)
```
