# 删除指定commit开始的文件

- git filter-repo 只能过滤某个分支的commit，无法指定或者范围
	-  `git filter-repo --force --path f3 --invert-paths --refs master`
- git rebase也无法做到，因为不能


# 查看2个commit之前的数量
`git rev-list --count 6b34ac51510516bd6a3ec2f5edc97413758d3ab1^..d8fa5ad85d476be6d60c96ace769a2c9293bc2a8`

# 丢弃已跟踪文件的改动 
git reset --hard HEAD 
# 同时删除未跟踪的文件和目录 
git clean -fd 
# 连 .gitignore 忽略的文件也删除 
git clean -fdx


# 仅看信息
- 仅拉指定分支的commit `git fetch repo branch`

# 删除指定commit之前的commit
- 先按照git-filter-repo
```bash
git filter-repo --force --commit-callback "
if commit.original_id == b'xxxxxxxxxxxxxxxxxxxxx':
    commit.parents = []
"
```

- 这个脚本不行，这个会仅保留该commit后的commit和文件，之前commit的文件会被完全删除，除非在保留的commit里，如果只是像删除这个commit记录，即恢复到仅fetch了该一个commit的情况，则不能这么做

# 局域网Clone
- 无需bare仓库，可直接 `git clone ssh://username@host/path/to/repo`
- 如果git仓库在windows，则必须用  `git clone [username@host|hostname]:[driver]:/path/to/repo`

# 查找指定commit创建时间后的tag

```powershell
$cutoff = [DateTimeOffset]::Parse((git show -s --format="%ci" xxx))
git for-each-ref --sort=creatordate --format="%(creatordate:iso)|%(refname:short)" refs/tags |
ForEach-Object {
    $parts = $_ -split '\|'
    [PSCustomObject]@{
        Date = [DateTimeOffset]::Parse($parts[0])
        Tag  = $parts[1]
    }
} | Where-Object { $_.Date -gt $cutoff } | Sort-Object Date
```
